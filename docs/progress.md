# 프로젝트 진행 과정

## 분석 (문제/지표/데이터 확정)

-   목표 정의
-   KPI 정의 + 계산 규칙 문서화
-   데이터 스키마 확정

> 산출물 : KPI 정의서, 데이터 스키마 + 샘플 50줄

## 설계 (아키텍처/화면/모듈 분리)

-   파이프라인 설계
-   화면 와이어프레임 (간단한 그림이면 충분)
-   구현 단위 쪼개기
    -   parser 모듈
    -   aggregator 모듈 (일/주/시간대 등)
    -   anomaly/rule (이상치: 음수 배달시간 등)
    -   frontend charts

> 산출물 : 폴더 구조 + API 응답 형태 (or 데이터 파일 포맷) + 화면 스케치

### 구현1 (데이터 파싱/정제/집계)

-   JS로 로그 파서 완성
-   집계 함수 구현 (시간대별, 지역별, 상태별)
-   "테스트 데이터로 결과가 맞는지" 콘솔 수준 확인

> 산출물 : 집계 JSON 생성기 / api용 데이터 1차

### 구현2 (대시보드 MVP)

-   차트 2~3개만 먼저 연결해서 MVP 완성
-   인터랙션 1개 이상 넣기
-   SCSS 적용 (레이아웃/테마) + 트윈맥스 (애니메이션은 최소로 "유의미한 곳만")

### 구현3 (고도화 + 품질)

-   이상치 처리/결측 처리 로직 추가
-   차트 추가 (분포/히트맵/상위 TOP N 등)
-   성능 : 집계는 미리 계산해서 내려주고, 프론트는 필터링 중심

> 산출물 : 완성형 대시보드 + 안정화된 집계

### 테스트/리팩토링/문서

-   단위 테스트 (최소)
    -   파싱이 깨지는 케이스
    -   시간 계산(타입존/음수/역전)
    -   상태 전이(주문->배차->픽업->완료)
-   리팩토링 (함수 이름/모듈 분리)
-   README : 실행 방법 + KPI 정의 + 스크린샷

> 산출물 : 테스트 + README + 코드 정리

### 배포 + 데모 시나리오

-   배포 (Vercel/Netlify + 간단한 API면 Render/Fly.io 등)
-   데모 스토리 작성 (3분)

> 산출물 : 배포 URL + 데모 시나리오
