# Overview

## 1. 프로젝트 개요

본 프로젝트는 **배달의민족(배민) 스타일의 배달 주문 흐름을 기반으로 한  
배달 주문 이벤트 로그 분석 시스템**을 구현하는 미니 프로젝트이다.

배달 주문 과정에서 발생하는 여러 이벤트를 로그 형태로 수집하고,
이를 가공·분석하여 **주문 리드타임, 병목 구간, 운영 KPI**를 계산하고
리포트 및 대시보드 형태로 시각화하는 것을 목표로 한다.

본 프로젝트는 **순수 JavaScript 기반**으로 구현되며,
실제 서비스 환경을 단순화한 **가상 이벤트 로그 데이터**를 사용한다.

---

## 2. 문제 정의 (Problem Statement)

배달 서비스 운영 과정에서는 다음과 같은 문제가 반복적으로 발생한다.

-   주문 지연이 발생했을 때 **어디에서 지연이 발생했는지 파악하기 어렵다**
-   조리 지연인지, 배차 지연인지, 라이더 이동 문제인지 **원인 구분이 명확하지 않다**
-   시간대·지역·가게별로 **운영 병목을 정량적으로 판단하기 어렵다**

이러한 문제는 대부분 **이벤트 단위 로그를 체계적으로 분석하지 못하기 때문에**
발생한다.

---

## 3. 프로젝트 목표 (Goals)

### G1. 주문 흐름을 이벤트 로그 기반으로 복원한다

-   하나의 주문(`order_id`)에 대해 발생한 이벤트를 시간 순서대로 재구성한다
-   이벤트 누락, 중복, 순서 오류가 발생하더라도 분석 가능한 형태로 정규화한다

---

### G2. 배달 리드타임과 병목 구간을 계산한다

-   주문부터 배달 완료까지의 **전체 리드타임**을 계산한다
-   주문 흐름을 여러 구간으로 분리하여 **구간별 소요 시간**을 계산한다
-   각 주문에서 **가장 오래 소요된 구간(병목 구간)**을 식별한다

---

### G3. 운영 의사결정에 활용 가능한 KPI를 산출한다

-   날짜, 시간대, 지역, 가게 단위로 KPI를 집계한다
-   지연 주문 비율, 평균 리드타임 등 핵심 지표를 계산한다
-   KPI 결과를 리포트 및 대시보드 형태로 시각화한다

---

## 4. 프로젝트 범위 (Scope)

### 포함 범위 (In Scope)

-   배달 주문 이벤트 로그 데이터 처리
-   주문 단위 타임라인 구성
-   KPI 계산 로직 구현
-   분석 결과 시각화 (웹 기반 대시보드)

### 제외 범위 (Out of Scope)

-   실시간 스트리밍 처리 (Kafka 등)
-   배달 시간 예측, 추천 시스템
-   결제, 정산, 개인정보 처리

---

## 5. 성공 기준 (Success Criteria)

본 프로젝트는 다음 조건을 만족할 경우 성공으로 판단한다.

-   샘플 데이터 기반으로 KPI가 **일관되게 재현**된다
-   일부 이벤트 누락 또는 비정상 순서가 존재하더라도 시스템이 중단되지 않는다
-   병목 구간 및 지연 원인을 **정량적으로 확인할 수 있다**
-   분석 결과를 한 화면에서 직관적으로 확인할 수 있다
